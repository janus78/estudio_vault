# 🧠 Mis Guías Técnicas

Este directorio contiene documentación profunda y frameworks de trabajo creados específicamente para la transición de **Senior Developer** a **Staff Engineer / Architect**. Las guías están optimizadas para el ecosistema **.NET/Azure**, pero cubren conceptos universales de arquitectura y diseño de sistemas.

## 🚀 Plan Maestro
- **[Roadmap Zero to Hero](./roadmap_zero_to_hero_updated.md)**: El plan de 26-30 semanas que orquesta todo el proceso de estudio, desde algoritmos hasta diseño de sistemas a gran escala.

## 🏗️ Arquitectura y Diseño de Sistemas
- **[System Design Framework](./guia_system_design_framework.md)**: Framework mental para resolver problemas de diseño de sistemas, incluyendo componentes, CAP Theorem y casos de estudio (Twitter, Uber, etc.).
- **[Arquitectura .NET y Patrones](./guia_dotnet_arquitectura_patrones_expanded.md)**: Profundización en SOLID, patrones GoF, Clean Architecture y DDD aplicado a .NET.
- **[Bases de Datos: Fundamentos a Producción](./guia_databases_fundamentos_produccion.md)**: Gestión de datos, optimización de índices, transacciones, replicación y sharding.

## 💻 Desarrollo y Frameworks
- **[Guía Definitiva ASP.NET Core](./guia_dotnet_entrevista_completo.md)**: Deep dive en el runtime de .NET, ciclo de vida de peticiones, middleware, DI y seguridad.
- **[Frontend para Arquitectos](./guia_frontend_arquitecto.md)**: Estrategias de renderizado (SSR, SSG, ISR), React avanzado, estado global y el rol estratégico de Blazor.
- **[Guía Maestra SD 1.5 + Forge + Backend .NET](./guia_sd15_forge_pipeline_backend_net_v3.md)**: Dominio de Stable Diffusion como sistema reproducible integrado en arquitecturas .NET.

## 🛠️ Operaciones, Seguridad y Observabilidad
- **[Observability, Security & DevOps](./guia_observability_security_devops.md)**: Guía práctica sobre los tres pilares de observabilidad, seguridad OWASP, CI/CD en Azure DevOps e Infraestructura como Código (IaC).

## 🤖 Optimización con IA y Herramientas
- **[Framework Personal de IA](./framework-ia-personal.md)**: Sistema de trabajo intencional y eficiente con ChatGPT, Gemini y Claude para ingenieros.
- **[Guía de Optimización Multi-IA](./guia-optimizacion-ia.md)**: Estrategias avanzadas para el uso combinado de ChatGPT, Claude y Gemini en flujos de ingeniería.
- **[Guía de IA No Técnica](./guia-ia-no-tecnica.docx)**: Conceptos y aplicaciones de IA explicados para perfiles no técnicos o visión de negocio.
- **[Aprovechamiento de Suscripciones IA](./guia-maxima-ia-subscripciones.md)**: Cómo exprimir los límites y capacidades de las versiones Plus/Pro para el trabajo técnico.
- **[NotebookLM para Staff Engineers](./guia_notebooklm_staff_engineer.md)**: Metodología para usar NotebookLM como motor de síntesis de conocimiento e integración con Obsidian.

_Nota: Estas guías son documentos vivos que se actualizan conforme avanza el proceso de aprendizaje y se descubren nuevos patrones en producción._
