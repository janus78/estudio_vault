# 🧠 Mis Guías Técnicas y Frameworks

Este repositorio es el núcleo de conocimiento estructurado para la transición hacia el rol de **Staff Engineer / Software Architect**. Aquí se consolidan frameworks de trabajo, guías de estudio profundo y estrategias de comunicación diseñadas para ingenieros de alto impacto.

## 📂 Estructura del Conocimiento

### 🤖 [01 - Guías Base IA](./01-Guias_Base_IA/)
*Maximización de la productividad mediante Inteligencia Artificial.*
- **Framework IA Personal**: Metodologías para integrar LLMs en el flujo diario.
- **Desarrollador Aumentado 2026**: Guía técnica para la ingeniería asistida por IA.
- **Optimización y Suscripciones**: Gestión estratégica de herramientas y modelos.

### 🗣️ [02 - Comunicación](./02-Comunicacion/)
*Habilidades blandas con enfoque técnico.*
- **Comunicación de Valor**: Cómo proyectar influencia y liderar decisiones técnicas como Senior/Staff Engineer.

### 🛠️ [03 - DotNet & Arquitectura](./03-DotNet_Recursos/)
*Profundización técnica y diseño de sistemas.*
- **Ecosistema .NET**: Arquitectura, patrones avanzados y preparación específica para entrevistas .NET.
- **System Design Framework**: Metodologías para diseñar sistemas escalables y resilientes.
- **Especializaciones**: Guías de Frontend para Arquitectos, Bases de Datos en Producción y Observabilidad/Seguridad.

### 🎓 [04.1 - Plan de Estudios (Staff)](./04_1-Plan_Estudios_Staff_Engineer/)
*El curriculum "Zero to Hero v2" completo.*
- **Módulos 1-7**: Desde Computer Science Fundamentals hasta Estrategias de Negociación y Entrevistas Staff.
- **Recursos**: Tracking de progreso, glosario técnico y bibliografía seleccionada.

### 🎒 [05 - Varios](./05-Varios/)
*Herramientas complementarias y roadmaps.*
- **NotebookLM**: Guía de uso para investigación técnica profunda.
- **Roadmap Global**: La visión completa del camino "Zero to Hero".
- **SD15 Forge**: Pipeline de backend especializado.

## 🧭 Cómo navegar este directorio
1. Si buscas un camino de estudio estructurado, comienza por el **[Plan de Estudios (04.1)](./04_1-Plan_Estudios_Staff_Engineer/)**.
2. Si necesitas resolver un problema técnico específico (Arquitectura, DBs, .NET), consulta **[DotNet & Arquitectura (03)](./03-DotNet_Recursos/)**.
3. Para optimizar tu flujo de trabajo con IA, revisa las **[Guías Base IA (01)](./01-Guias_Base_IA/)**.
