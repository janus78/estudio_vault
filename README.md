# Estudio Vault

Este repositorio es un espacio personal centralizado para la organización de materiales de estudio, guías técnicas y preparación para entrevistas. Está diseñado para ser utilizado como una base de conocimiento en Obsidian.

## 📂 Estructura del Vault

El contenido se organiza en las siguientes secciones principales:

- **[00 - Inbox](./00%20-%20Inbox/)**: Punto de entrada para capturas rápidas, ideas sueltas o material que aún no ha sido clasificado.
- **[01 - Conceptos](./01%20-%20Conceptos/)**: Notas detalladas y referencias sobre conceptos teóricos de ingeniería de software.
- **[02 - Entrevistas](./02%20-%20Entrevistas/)**: Recursos específicos para la preparación de procesos de selección, incluyendo preguntas típicas y estrategias de resolución.
- **[03 - System_Design](./03%20-%20System_Design/)**: Material enfocado en el diseño de sistemas a gran escala, escalabilidad y patrones arquitectónicos.
- **[04 - Mis_Guias](./04%20-%20Mis_Guias/)**: Documentación técnica detallada y guías de referencia rápida creadas durante el proceso de estudio.
- **[05 - Templates](./05%20-%20Templates/)**: Plantillas reutilizables para mantener la consistencia en las notas y estructuras del vault. Incluye prompts avanzados para la gestión de contexto en sesiones con IA.

## 🚀 Contenido Destacado (Mis Guías)

Dentro de la sección de guías personales, se encuentran materiales sobre:

- **Arquitectura y Patrones:** .NET avanzado, patrones de diseño y arquitectura de software.
- **Infraestructura y DevOps:** Observabilidad, seguridad y fundamentos de bases de datos en producción.
- **Frontend:** Guía para arquitectos frontend.
- **Carrera y Rol:** Guías para Staff Engineer y roadmaps de aprendizaje (Zero to Hero).
- **Inteligencia Artificial:** Optimización y gestión de suscripciones de IA.
- **System Design:** Frameworks específicos para abordar problemas de diseño.

## 🛠️ Uso

Este vault está optimizado para su uso con **Obsidian**, aprovechando los enlaces internos y la visualización de gráficos. Cada carpeta contiene su propio `README.md` para facilitar la navegación y el mantenimiento independiente.
