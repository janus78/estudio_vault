# Estudio Vault

Este repositorio es un espacio personal centralizado para la organización de materiales de estudio, guías técnicas y preparación para entrevistas. Está diseñado para ser utilizado como una base de conocimiento en Obsidian.

## 📂 Estructura del Vault

El contenido se organiza en las siguientes secciones principales:

- **[00 - Inbox](./00%20-%20Inbox/)**: Punto de entrada para capturas rápidas, ideas sueltas o material de investigación pendiente de clasificar.
- **[01 - Conceptos](./01%20-%20Conceptos/)**: Notas detalladas y referencias sobre conceptos teóricos de ingeniería de software.
- **[02 - Entrevistas](./02%20-%20Entrevistas/)**: Recursos específicos para la preparación de procesos de selección.
- **[03 - System_Design](./03%20-%20System_Design/)**: Material enfocado en el diseño de sistemas a gran escala y arquitectura.
- **[04 - Mis_Guias](./04%20-%20Mis_Guias/)**: Documentación técnica organizada por categorías (IA, .NET, Comunicación, Staff Engineer).
- **[05 - Templates](./05%20-%20Templates/)**: Plantillas reutilizables para notas y prompts avanzados de IA.
- **[06 - Proyectos](./06%20-%20Proyectos/)**: Casos prácticos e implementaciones reales para aplicar el conocimiento.

## 🚀 Contenido Destacado

- **Staff Engineer Path:** Roadmaps detallados y 6 módulos completos que van desde fundamentos CS hasta IA Avanzada.
- **IA Workflow:** Framework personal de IA, agentes inteligentes y optimización de suscripciones.
- **Arquitectura:** Inmersión profunda en .NET, System Design, Observabilidad y Patrones Distribuidos.

## 🛠️ Uso

Este vault está optimizado para su uso con **Obsidian**. Cada carpeta contiene su propio `README.md` para facilitar la navegación independiente.
